# Sentrix V7 — API Reference

Sentrix V7 unifies all engine APIs into a single FastAPI service available on port `8000`. All endpoints are prefixed with `/api/v1/`.

## Authentication

Authentication can be enforced globally by setting `SENTRIX_AUTH_ENABLED=true` in your `.env`. 
When enabled, you must provide either:
- A raw API Key in the `X-API-Key` HTTP Header.
- A JWT Token in the `Authorization: Bearer <token>` HTTP Header.

### Roles
- **Admin**: Has access to all endpoints.
- **SOC Analyst**: Can view and trigger alerts, investigations, and forecasts.
- **Read Only**: Can only view data, cannot write or trigger.

## 1. Threat Engine (`/api/v1/threat/`)

| Method | Endpoint | Description | Role Required |
|---|---|---|---|
| POST | `/events/ingest` | Ingests a raw event payload for threat analysis | SOC Analyst |
| GET | `/status` | Retrieve the threat engine's current operational status | SOC Analyst |

## 2. Prediction Engine (`/api/v1/predictions/`)

| Method | Endpoint | Description | Role Required |
|---|---|---|---|
| GET | `/forecast` | Get live prediction forecasts for active attack paths | SOC Analyst |
| GET | `/forecast/{ip}` | Get prediction forecast for a specific IP | SOC Analyst |
| GET | `/history` | Get historical predictions | SOC Analyst |

## 3. Investigation Engine (`/api/v1/investigations/`)

| Method | Endpoint | Description | Role Required |
|---|---|---|---|
| POST | `/generate/{ip}`| Trigger a background investigation for an IP | SOC Analyst |
| GET | `/incidents` | List all incidents generated by the investigation engine | SOC Analyst |

## 4. Rule Define Studio (`/api/v1/rules/`)

| Method | Endpoint | Description | Role Required |
|---|---|---|---|
| GET | `/` | List all available active rules | Admin / SOC Analyst |
| POST | `/` | Create a new custom rule | Admin |
| GET | `/{id}` | Retrieve a specific rule | Admin / SOC Analyst |
| PUT | `/{id}` | Update an existing rule | Admin |
| DELETE| `/{id}` | Delete / Archive a rule | Admin |
| POST | `/test` | Test a rule against a sample event | Admin / SOC Analyst |

## 5. Connector Framework (`/api/v1/connectors/`)

| Method | Endpoint | Description | Role Required |
|---|---|---|---|
| GET | `/` | List all available connectors and their enabled status | Admin / SOC Analyst |
| POST | `/{name}/enable`| Enable a specific connector | Admin |
| POST | `/{name}/disable`| Disable a specific connector | Admin |
| POST | `/enrich/{ip}`| Enrich an IP address using all enabled connectors | Admin / SOC Analyst |

## 6. Admin & Health (`/api/v1/admin/`)

| Method | Endpoint | Description | Role Required |
|---|---|---|---|
| GET | `/health` | Returns the health status of all internal engines | None (Open) |
| GET | `/ready` | Readiness probe | None (Open) |
| POST | `/auth/token` | Exchange a valid X-API-Key for a JWT Token | Admin |
